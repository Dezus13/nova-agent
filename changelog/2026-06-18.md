## TS07: User-Owned Workflow Operations

- Создан `docs/specs/технические/07-user-owned-workflow-operations.md`.
- Зафиксирована операционная модель пользователя: разрешённые и запрещённые операции для всех User-Owned сущностей (Action Plan, Step Progress, User Open Question, User Note, Checked Source Mark, History Events).
- Закрыты все открытые lifecycle-решения из TS02 и TS03 (8 MVP-решений).

### Принятые MVP-решения

- **Решение 1:** Checked Source Mark не является Plan-Creating Stateful Action. Создаётся только внутри существующего active Action Plan.
- **Решение 2:** Plan-Creating Stateful Actions — исчерпывающий список: Explicit Start Plan, Change Step Progress Status, Create User Open Question, Create User Note.
- **Решение 3:** User Open Question status `irrelevant` является финальным состоянием. Переход из `irrelevant` запрещён; новая неопределённость = новый вопрос.
- **Решение 4:** Action Plan имеет только два состояния в MVP: `active` и `completed`. Состояния `archived`, `hidden`, `deleted_by_user`, `reopened`, `restored` не входят в MVP.
- **Решение 5:** Completed Action Plan не переоткрывается. Переход `completed → active` запрещён.
- **Решение 6:** Новый проход по сценарию создаёт новый Action Plan (новую сущность). Старый completed план не изменяется.
- **Решение 7:** Один active Action Plan на User на stable Scenario identity одновременно. Partial rule: только active планы входят в uniqueness.
- **Решение 8:** RLS Pattern B распространяется на active И completed планы. Уточняет TS05 Amendment 03.

### Зафиксированные правила

- Полный граф переходов Step Progress (15 допустимых; возврат в `not_started` запрещён).
- Полный граф переходов User Open Question (8 допустимых; `irrelevant` финальный; `clarified_by_user → requires_check` допустим).
- User Note lifecycle: `created`, `edited_by_user`, `hidden_by_user`/`deleted_by_user`; каждая операция создаёт History Event.
- Минимальный payload всех 11 event_types History Event зафиксирован в таблице.
- History Events: append-only инвариант; не редактируются, не удаляются.
- Scenario Version Lock: Action Plan всегда связан с версией момента создания; контент не мигрирует автоматически.
- Pattern B уточнена: применяется к active и completed планам (доступ к deprecated/superseded Scenario Version).
- 22 инварианта, 16 рисков, 8 решений перед TS08 и критерии приёмки зафиксированы.
