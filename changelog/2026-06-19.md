# Changelog 2026-06-19

## UI-01: UI Design Rules

- Создан `docs/specs/ui/01-ui-design-rules.md`.
- Зафиксированы архитектурные правила интерфейса Nova Agent.

### Назначение документа

UI-01 является архитектурным слоем между операционными спецификациями (TS06, TS07) и реализацией. Документ не является дизайн-системой и не выбирает цвета, шрифты, пиксели, компоненты, CSS или анимации.

### Содержание UI-01

**Роли и доступ (visibility rules):**
- User видит только published Scenario Versions (+ deprecated/superseded при Pattern B).
- User видит только собственные User-Owned Data.
- Content Admin видит весь Content Model (включая draft), но не видит User-Owned Data.
- Изоляция ролей полная: нет экрана, доступного обеим ролям.

**Концептуальные экраны пользователя (9 экранов):**
- U-01: Каталог жизненных ситуаций.
- U-02: Карточка жизненной ситуации.
- U-03: Просмотр сценария (без плана — read-only).
- U-04: Мои планы действий.
- U-05: Детальный просмотр плана.
- U-06: Детальный просмотр шага внутри плана.
- U-07: История плана.
- U-08: Открытые вопросы плана.
- U-09: Просмотр устаревшего сценария (Pattern B).

**Концептуальные экраны Content Admin (7 экранов):**
- A-01–A-07: управление жизненными ситуациями, сценариями, черновиками, версиями, источниками, Template OQ, предупреждениями.

**Ключевые архитектурные принципы:**
- Контекст важнее действия: Warnings и Restrictions показываются до CTA и контролов действий.
- Nova Agent — не Task Manager: запрещены приоритеты, дедлайны, исполнители, канбан, спринты.
- UI обязан отражать реальные границы продукта: Progress = личная отметка; UOQ = личный вопрос; History = внутренние события Nova Agent.
- Строгое разделение: Scenario vs. Action Plan; Template OQ vs. User OQ; Source vs. Checked Source Mark; Content Model vs. User-Owned Data.

**22 UI-инварианта, включая:**
- Просмотр сценария не создаёт Action Plan.
- Отметка Source не является plan-creating action.
- Каждый Progress-статус маркирован «Ваша отметка».
- Completed Action Plan: Progress read-only.
- Pattern B version warning баннер обязателен.
- Boundary disclaimer постоянен на всех plan-экранах.
- Переход Progress → `not_started` скрыт в UI.
- Переход UOQ из `irrelevant` недоступен.

**Запрещённые UI-паттерны:** 19 явно перечисленных паттернов, включая task-manager элементы, официально звучащие статусы, смешение Template OQ и User OQ, показ draft Scenario Version пользователю.

**10 рисков** с источниками и правилами снижения.

**Решения до TS08 (10 пунктов):** инвентарь экранов, visibility rules, Plan-creation trigger, Pattern B signal, User Note FK, Completed Plan read-only.

**Решения, отложенные до TS08 (10 пунктов):** pagination, optimistic UI, auth flow, routing, hidden Note, CSM uniqueness.
