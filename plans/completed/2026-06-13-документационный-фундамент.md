# Документационный фундамент Nova Agent

## Цель этапа

Зафиксировать базовую документацию проекта до перехода к функциональным спецификациям и реализации.

## Область работ

- Синхронизировать `AGENTS.md` с иерархией документации из `docs/глобальная-спецификация.md`.
- Создать `docs/карта-функций.md`.
- Создать `docs/пользовательские-сценарии.md`.
- Подготовить `docs/roadmap.md`.
- Подготовить `docs/техническая-архитектура.md`.
- После завершения базовой документации перейти к функциональным спецификациям в `docs/specs/функции`.

## Ограничения

- Не писать код.
- Не создавать `src`.
- Не создавать `app`.
- Не создавать `components`.
- Не создавать `api`.
- Не создавать `database`.
- Не создавать документы или папки вне текущей структуры без отдельного решения.

## Порядок выполнения

1. Синхронизировать `AGENTS.md`.
2. Создать карту функций.
3. Создать пользовательские сценарии.
4. Подготовить roadmap.
5. Подготовить техническую архитектуру.
6. Только после этого переходить к функциональным specs.

## Критерии завершения

- `AGENTS.md` не противоречит `docs/глобальная-спецификация.md`.
- `docs/карта-функций.md` описывает продуктовые функции без технической реализации.
- `docs/пользовательские-сценарии.md` описывает базовые сценарии пользователя.
- `docs/roadmap.md` задаёт последовательность этапов.
- `docs/техническая-архитектура.md` фиксирует архитектурные принципы без выбора стека, если стек ещё не утверждён.
- Функциональные specs не создаются до завершения базовых документов.
