## Scope этапа

### TS07: User-Owned Workflow Operations

Следующий технический документ. Операционная модель обычного пользователя: разрешённые и запрещённые операции с Action Plan, Step Progress, User Open Question, User Note, Checked Source Mark и History Events.

TS07 является операционным аналогом TS06 для стороны пользователя. Без TS07 нельзя писать API Spec: API объединяет admin-операции (TS06) и user-операции (TS07).

### UI-01: UI Design Rules

Дизайн-принципы и ключевые правила интерфейса. Может выполняться параллельно с TS07. Не должен предшествовать TS07: UI строится на операционных инвариантах, а не наоборот.

### TS08: API Layer Specification

Описание API-поверхности: операции, параметры, ответы, auth. Может быть подготовлен только после TS07 и UI-01, поскольку API объединяет обе стороны системы.

### MVP Work Plan

Технический план реализации с порядком задач, зависимостями и критериями готовности. Выполняется после TS07, UI-01, TS08.

### MVP Seed Content Plan

Описание первого набора жизненных ситуаций, сценариев, шагов, документов и источников для австрийского контекста MVP. Может выполняться параллельно с техническими specs.

### README Navigation Polish

Добавление навигационного содержания в README.md перед передачей проекта. Выполняется последним, после стабилизации структуры всех документов.

## Порядок выполнения

## Критерии завершения этапа

Этап считается завершённым, если:

- TS07 готов и не противоречит TS01–TS06, функциональным specs и AGENTS.md.
- UI-01 готов и не выходит за пределы операционных инвариантов TS07.
- TS08 готов и объединяет операционные модели TS06 и TS07.
- MVP Work Plan готов и содержит порядок задач с зависимостями.
- MVP Seed Content Plan готов и описывает первый ограниченный набор австрийских сценариев.
- Код не написан до завершения всех перечисленных документов.
- Структура проекта не засорена лишними файлами или папками.
